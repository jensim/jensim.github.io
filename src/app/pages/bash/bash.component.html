BASH

Bash is versatile, and it's everywhere.
And even though the default terminal for my Mac is zsh nowadays, any time I write a script, it starts with #!/bin/bash.
There are some reasons, one is the reason that you are able to export functions. (Support currently lacking in zsh)
With it, it's a lot simpler to run things in parallel with parallel or xargs against the exported func..
At work I've built a closed source homebrew tap where I distribute scripts, to enable people in their daily work.
  • Execute maven version upgrades for parent, properties, and dependencies. Parameterized with version rules and so on.
  • Generate MarkDown tables containing pre-baked links to all the build-, log-, vcs-systems, and landing pages in all envs for the current repo.
  • Code migrations.
    • When we switched from one build system to another
    • When there where major changes to swagger annotations.
    • JUnit4 to JUnit5
